Project Overview
Build a mobile-first web application for the 2026 state primary elections in 5 pilot states (NY, NJ, PA, CT, TX). Core MVP focuses on ballot information, voter card generation, and social sharing.

PHASE 3: LLM-Powered Content Processing
Task 3.1: Create Ballot Simplification API
javascriptBuild API endpoint that:

POST /api/simplify-ballot-measure
Input: {
  original_text: string,
  measure_id: string
}

Processing:
1. Call Claude API with prompt:
   "You are helping young voters understand ballot measures.
    Convert this legal text into 3 versions:
    1. One sentence (max 15 words)
    2. Simple explanation (max 50 words, 8th grade reading level)
    3. Detailed breakdown (max 200 words)
    
    Include:
    - What changes
    - Who it affects
    - Estimated cost/benefit
    - Why it matters to young adults
    
    Original text: [original_text]"

2. Validate response structure
3. Cache results
4. Return JSON with all versions
Task 3.2: Build Bias Detection System
javascriptCreate bias checker:

POST /api/check-bias
Input: {
  content: string,
  content_type: 'summary' | 'argument'
}

Processing:
1. Check for loaded language
2. Verify both sides represented
3. Flag missing context
4. Return bias score (0-100) with specific issues

If score > 70:
- Trigger human review
- Add "under review" label
- Suggest edits
